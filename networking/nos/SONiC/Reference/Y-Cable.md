# Y-Cable



## Y Cable 概述
**Y Cable = Y型线缆 / Mux Cable（复用器线缆）**
它是一种**带智能芯片、内置MUX（多路复用器）的有源高速线缆**，外形像字母 **Y**：
- **一端**：接服务器的 **NIC（网卡）**
- **另外两端**：分别接两台交换机 **TOR A / TOR B**

核心作用：**把服务器的一路物理链路，自动/手动切换到两台交换机中的任意一台**，实现**链路冗余、高可用**。

简单说：
**普通光模块是“一对一”，Y Cable 是“一对二”的智能切换线缆。**

---

## Y Cable 的核心特征
从 SONiC `src/sonic-platform-common/sonic_y_cable/y_cable.py` 可以看出它的**全部关键能力**：

### 1. 内置硬件MUX（核心）
- 可以**手动强制切换**到 TOR A / TOR B
- 可以**自动切换**：主链路断了自动切到备用链路
- 同一时间**只选一条链路转发**，不会同时通

### 2. 可监控状态
代码提供大量查询API：
- 当前MUX指向哪台交换机
- 哪条链路真正在**转发流量（Active）**
- NIC / TOR A / TOR B 三方链路是否UP
- 切换次数统计（手动/自动）

### 3. 自带诊断功能
- **BER 误码率检测**
- **Eye Diagram 眼图测量**
- **PRBS 测试模式**
- **环回测试（Loopback）**
- 温度、电压监控（内部/NIC端）

### 4. 可配置、可升级
- 支持**固件升级/激活/回滚**
- 支持均衡参数（Cursor）读取
- 支持 lanes 通道状态检测

### 5. 工作在SONiC交换机系统
- 通过 **EEPROM 寄存器** 控制硬件
- 属于 **SFP/光口驱动层** 的扩展功能

---

## Y Cable 和 普通光模块 的核心区别

| 对比项 | 普通光模块（SFP/SFP+/QSFP） | Y Cable（Y型智能线缆） |
|--------|------------------------------|------------------------|
| **连接结构** | 一对一：交换机 ↔ 服务器 | 一对二：1个NIC ↔ 2台交换机 |
| **核心硬件** | 光电转换芯片 | **MUX复用器 + 智能控制芯片** |
| **链路能力** | 单链路，无冗余 | 双链路冗余，支持主备切换 |
| **切换方式** | 必须靠软件/协议切换（如LACP、BOND） | **硬件级秒级切换**，不依赖操作系统 |
| **功能** | 只负责光电转换、透传流量 | 流量选路、状态检测、故障自愈 |
| **控制方式** | 只读信息，无法控制链路 | 可写寄存器，控制MUX、模式、测试 |
| **使用场景** | 普通数据传输 | 高可用服务器、需冗余链路的场景 |
| **形态** | 模块+光纤 | 一体化有源线缆 |

---

总结:

1. **Y Cable 是带MUX的智能Y型线缆**，一端接网卡，另外两端接两台交换机。
2. **它的核心能力是硬件级主备链路切换**，实现高可用、故障自动自愈。
3. **普通光模块只做光电转换，Y Cable 多了选路、控制、诊断、冗余能力**。

---

### 总结
- **Y Cable**：**有源智能冗余线缆**，1分2，硬件切换，高可用专用
- **光模块**：**光电转换器件**，1对1，只负责传输
- SONiC 系统存在用来**控制Y Cable的MUX、读取状态、做诊断、升级固件**的驱动接口: `src/sonic-platform-common/sonic_y_cable/y_cable_vendor_mapping.py`
