# 设备资源访问


## Summary












## Detail

### /dev/mem - 直接读写内存

`/dev/mem` 是**物理内存的“镜像文件”**，读写该文件即读写内存，打开文件的offset即是内存地址(如8G/16G)的偏移。

- 你用 `mmap` 时传入的 `offset`：是**物理地址**。比如 `offset = 0x10000000` 就对应物理内存的 0x10000000 位置。
- 你用 `pread` / `pwrite` 或 `lseek` + `read` / `write` 时：**文件偏移量**即是**物理地址**。
- `mmap` 成功后返回给你的指针（`void *`）：是**进程的虚拟地址**。CPU 通过页表把这个虚拟地址翻译到上面那个物理地址去访问。

> 传给 `/dev/mem` 的是物理地址，拿到手用的文件/dev/mem的操作指针是虚拟地址。



### /proc/iomem - 查看给IO设备的物理地址空间分配

`/proc/iomem` 记录了**物理内存地址范围（RAM + MMIO）分配情况**，可查看哪些物理地址被哪个设备/驱动占用。反映了 CPU 可访问的物理内存和 I/O 区域的布局，包括硬件设备的寄存器、保留内存区域等。

- 该文件列出了系统当前所有的 内存映射 I/O（MMIO）区域 的物理地址范围，例如：

  - PCI 总线地址空间
  - 板载外设寄存器
  - 显存、ACPI、系统 ROM 等

- 示例：

```shell
$ cat /proc/iomem

00001000-0009fbff : System RAM
...
feb00000-feb03fff : 0000:00:02.0
  feb00000-feb03fff : i915
fec00000-fec003ff : IOAPIC 0

  fed1f410-fed1f414 : iTCO_wdt.0.auto
    fed1f410-fed1f414 : iTCO_wdt.0.auto

feffc000-feffffff : Reserved

```


### /proc/ioports - 查看给IO设备的端口地址空间分配

`/proc/ioports` 的内容是系统当前**设备 I/O 端口地址空间的分配情况**，X86下与`/dev/mem`分开；而ARM下`/proc/ioports`使用的地址是内存物理地址。

- X86：
  - 作用：它和`/proc/iomem`是互补的，iomem 是内存映射 I/O，用 mov 指令访问；ioports 是独立 I/O 空间，在 X86 上通过 in/out 指令访问。
  - 内核支持：需要内核开启 CONFIG_DEVPORT（通常默认开启）。检查是否存在 `/dev/port`。
  - 权限：读写 I/O 端口需要 CAP_SYS_RAWIO 能力，直接使用 root 或 sudo 执行。
  - 风险提示：直接操作 I/O 端口可能导致系统死锁或硬件损坏，请确保地址正确。

  - 常见的设备端口及其对应的设备类型：
    
    | 端口范围                  | 硬件                 |
    | :------------------------ | :------------------- |
    | `0020-0021`               | 8259 PIC 主片        |
    | `0060`/`0064`             | 键盘控制器（8042）   |
    | `03f8-03ff`               | 串口 COM1            |
    | `0cf8-0cff`               | PCI 配置空间访问端口 |
    | `0170-0177` / `01f0-01f7` | IDE/SATA 传统端口    |
    | `03c0-03df`               | VGA 显卡             |

  - 示例：

    ```shell
    $ cat /proc/ioports
    # I/O端口地址范围 : 注册该区域的名字，通常是驱动或设备名
    0000-001f : dma1
    0020-0021 : pic1
    0040-0043 : timer0
    0050-0053 : timer1
    0060-0060 : keyboard
    0064-0064 : keyboard
    0070-0071 : rtc_cmos
    0080-008f : dma page reg
    00a0-00a1 : pic2
    00c0-00df : dma2
    00f0-00ff : fpu
    0170-0177 : ata_piix
    01f0-01f7 : ata_piix
    0376-0376 : ata_piix
    03c0-03df : vga+
    03f6-03f6 : ata_piix
    03f8-03ff : serial
    0400-047f : pnp 00:01
        0400-0403 : ACPI PM1a_EVT_BLK
        0404-0405 : ACPI PM1a_CNT_BLK
        0408-040b : ACPI PM_TMR
    0500-057f : pnp 00:01
    0cf8-0cff : PCI conf1
    ...
    d000-dfff : PCI Bus 0000:01
        d000-d0ff : 0000:01:00.0
        d000-d0ff : e1000
    ...

    ```

  - 使用方法：需配合`/dev/port`使用
    - C语言
      ```c
      #include <stdio.h>
      #include <fcntl.h>
      #include <unistd.h>
      #include <stdint.h>

      int main() {
          int fd = open("/dev/port", O_RDWR);
          if (fd < 0) { perror("open"); return 1; }
          // 0x3f8 来源于 cat /proc/ioports -> 03f8-03ff : serial
          // 把文件偏移定位到端口 0x3f8（串口COM1基址）
          lseek(fd, 0x3f8, SEEK_SET);
          // 读取端口的一个字节
          uint8_t val;
          read(fd, &val, 1);
          printf("port 0x3f8: 0x%02x\n", val);
          // 写入端口
          val = 0x0C;
          write(fd, &val, 1);
          close(fd);
          return 0;
      }

      // 方法2, 不推荐
      #include <stdio.h>
      #include <sys/io.h>
      #include <unistd.h>

      int main() {
          // 获取权限
          if (ioperm(0x3f8, 8, 1)) {
              perror("ioperm");
              return 1;
          }

          // 直接读写端口
          unsigned char val = inb(0x3f8);
          printf("0x3f8: 0x%02x\n", val);
          outb(0x0C, 0x3f8);

          // 释放权限
          ioperm(0x3f8, 8, 0);
          return 0;
      }
      ```
    - Linux原生命令：
      ```shell
      # 读
      ## 使用 dd + hexdump 输出十六进制值
      sudo dd if=/dev/port bs=1 count=1 skip=$((0x3F8)) status=none | hexdump -e '"%02x\n"'

      ## 使用 dd + od
      sudo dd if=/dev/port bs=1 count=1 skip=$((0x3F8)) status=none | od -An -tx1 | tr -d ' '

      ## 连续读
      sudo dd if=/dev/port bs=1 count=8 skip=$((0x3F8)) status=none | hexdump -C

      # 写
      ## 写单字节
      printf '\x48' | sudo dd of=/dev/port bs=1 count=1 seek=$((0x3F8)) status=none

      ## 将 3 字节写入起始端口 0x378（并口数据端口）
      printf '\x55\xaa\xff' | sudo dd of=/dev/port bs=1 count=3 seek=$((0x378)) status=none
      ```


- ARM：
  - 作用：在 ARM 架构下，没有 x86 那样的独立 I/O 端口地址空间。所有外设寄存器都是内存映射 I/O（MMIO），即“端口”实际就是一段物理内存地址。因此，Linux 下用纯命令读写 ARM I/O 端口，等同于通过 /dev/mem 读写物理地址。
  - 必须 root：访问 /dev/mem 需要 CAP_SYS_RAWIO 权限。
  - 地址对齐：某些外设要求 32 位对齐访问，使用 bs=4 按 4 字节读写。
  - 大小端：ARM 通常为小端序，多字节写入时要注意字节顺序。


  - 示例：

    ```shell
    $ cat /proc/ioports

    0000-001f : dma1
    0020-0021 : pic1
    0040-0043 : timer0
    0050-0053 : timer1
    0060-0060 : keyboard
    0064-0064 : keyboard
    0070-0071 : rtc_cmos
    0080-008f : dma page reg
    00a0-00a1 : pic2
    00c0-00df : dma2
    00f0-00ff : fpu
    0170-0177 : ata_piix
    01f0-01f7 : ata_piix
    0376-0376 : ata_piix
    03c0-03df : vga+
    03f6-03f6 : ata_piix
    03f8-03ff : serial
    0400-047f : pnp 00:01
        0400-0403 : ACPI PM1a_EVT_BLK
        0404-0405 : ACPI PM1a_CNT_BLK
        0408-040b : ACPI PM_TMR
    0500-057f : pnp 00:01
    0cf8-0cff : PCI conf1
    ...
    d000-dfff : PCI Bus 0000:01
        d000-d0ff : 0000:01:00.0
        d000-d0ff : e1000
    ...

    ```

  - 使用方法：需配合`/dev/mem`使用
    - C语言
      ```c
      #include <stdio.h>
      #include <fcntl.h>
      #include <unistd.h>
      #include <stdint.h>

      int main() {
          int fd = open("/dev/mem", O_RDWR);
          if (fd < 0) { perror("open"); return 1; }
          // 0x000003f8 来源于 cat /proc/ioports -> 03f8-03ff : serial
          lseek(fd, 0x000003f8, SEEK_SET);
          uint8_t val;
          read(fd, &val, 1);
          printf("port 0x000003f8: 0x%02x\n", val);
          val = 0x0C;
          write(fd, &val, 1);
          close(fd);
          return 0;
      }
      ```
    - Linux原生命令：
      ```shell
      # 读
      # 读取 4 字节，每字节偏移增加 (以树莓派 3/4 的 GPIO 寄存器基址 0x3F200000 为例，读取 4 字节)
      sudo dd if=/dev/mem bs=1 count=4 skip=$((0x3F200000)) status=none | hexdump -C

      ## 使用 dd + od
      sudo dd if=/dev/mem bs=1 count=4 skip=$((0x3F200000)) status=none | od -A x -t x4

      ## 连续读
      sudo dd if=/dev/mem bs=4 count=1 skip=$((0x3F200000/4)) status=none | hexdump -e '/4 "%08x\n"'

      # 写
      ## 写单字节
      sudo dd if=/dev/mem bs=1 count=4 skip=$((0x3F200000)) status=none | hexdump -C
      sudo dd if=/dev/mem bs=1 count=4 skip=$((0x3F200000)) status=none | od -A x -t x4

      ## 将 假设要写入值 0x00112233，在内存中的顺序是 0x33 0x22 0x11 0x00 (写入 32 位字（小端序）)
      printf '\x33\x22\x11\x00' | sudo dd of=/dev/mem bs=4 count=1 seek=$((0x3F200000/4)) status=none
      ```
